About Heatify

Version 0.2.2 [BETA]

The main purpose of Heatify is to encourage good Agile development practice, where contributors make small and frequent commits. Heatify accomplishes this by conveniently showing developers and managers their project activity. Heatify is also a fun way to compete for "heat" on popular projects, and compare the activity of entire squads on their project's.

Heatify works by keeping track of Git repositories that are entered in by users. Under the hood, Heatify clones repositories, analyses their activity, and keeps new data in-sync with a Cloudant database. By nature, viewing of a Heatmap for a brand-new repository can take a bit longer to load, depending on how large the repository is. This is because Heatify must clone the repository for the first time.

Heatify is developed and maintained by Brandon on the Dashboard squad of IBM Alchemy, with much of the high-level design coming from Andrew (Roo) Low, our squads STSM.

Usage

Heatify currently supports repositories from Github, IBM GHE, IBM GitLab, and IBM DevOps Services Here are some of the things you can do with Heatify:

- Public Repositories: easily render a public repo into a Heatmap. Simply copy and paste the HTTP or HTTPS URL (or for IBM GHE, the SSH link) into the Git Url field on the Landing Page

- Private Repositores: first add the Git-Monitor Bot as a member to your private repo. NOTE: private repositories are temporarily unavailable while we re-configure Git-Monitor's accounts.

- Squad Heatmaps: View a Heatmap showing an entire squad's activity on all of their repositories. For this feature, you must submit a JSON config file containing a list of a Squads members, and repositories. Please contact Brandon at schurman@ca.ibm.com or message @brandonsc on Slack.

- User Heatmaps: coming soon! you can view a Heatmap showing your activity on all repositories you track on Heatify.
