About Heatify

Version 0.1 [BETA]

Heatify is developed by the Reference-Apps squad of the IBM Alchemy project. It's main purpose is to encourage good Agile development practice, where contributors make small and frequent commits. Heatify accomplishes this by showing developers, quickly and visually, their project activity and commits are.

The Reference-Apps team develops applications which test our IBM Bluemix cloud platform, as well as teach our customers about the advantages of using Bluemix and how to use it.

Haetify is our squad's first reference-app written in Go.

Usage

Heatify currently supports both GitHub and IBM Jazz git repositories. Here are some of the things you can do with Heatify.

- Public Repositories: easily render a public repo into a HeatMap. Simply copy and paste the HTTP or HTTPS URL into the field on the Landing Page

- Private Repositores: first add the Git Monitr bot as a member to your private repo. Heatify's is becoming increasingly more secure, but for now becareful if you have code you don't want out in the world.

- Repository HeatMap: By default, Heatify will generate a HeatMap showing how many commits are made on each day of the month. The HeatMeter shows how many commits qualify for each tier.

- User HeatMap: coming soon! you can view a HeatMap showing your activity on all repositories you track on Heatify.
